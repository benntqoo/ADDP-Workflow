遗留项目**智能接入**向导：

1. 扫描项目结构和技术栈
2. 分析代码规模和质量
3. 识别现有的编码规范
4. 检查现有文档分布情况
5. 生成项目健康度报告
6. 创建适配性的 CLAUDE.md

生成的 CLAUDE.md 必须包含：
- 现有项目的技术债务和改进计划
- 渐进式重构策略
- 文档规范化计划（将散落文档迁移到 docs/ 目录）
- 保持向后兼容的开发约定

文档管理规范：
- 新文档必须放在 docs/ 相应子目录
- 逐步迁移旧文档到标准位置
- API 文档 → docs/api/
- 架构文档 → docs/architecture/
- 其他按类别归档

注：这是遗留项目，请保持谦逊和谨慎，尊重历史决策。
